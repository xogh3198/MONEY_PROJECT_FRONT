# InvestBoard 실험 기록

마지막 업데이트: 2026-07-24

## 작성 규칙

모든 제품·콘텐츠 실험은 구현 전에 아래 항목을 채운다.

- 실험 ID / 상태 / 담당
- 문제와 대상 사용자
- 근거와 확인일
- 가설
- 구현 범위와 제외 범위
- 선행 지표, 수익 지표, 비용
- 성공·중단 기준과 검토일
- 정책·법률·보안·저작권 위험
- 외부 실행 승인 지점
- 결과와 다음 결정

## E-001 — 데이터 기반 콘텐츠 스튜디오 MVP

- 상태: 개발 중
- 시작일: 2026-07-22
- 대상: InvestBoard 운영자
- 문제: 수집된 뉴스와 시장 데이터가 콘텐츠 제작 우선순위와 고유한 쇼츠 대본으로 연결되지 않는다.
- 근거: 뉴스·시장지표 수집은 운영 중이지만 수집 결과를 제작 후보로 점수화하거나 구조화된 초안으로 만드는 기능이 없다.
- 가설: 금융 관련성·최신성·인기 신호로 후보를 정렬하고 45~60초 초안을 생성하면, 후보 탐색과 초안 작성 시간을 건당 30분 이상에서 10분 이하로 줄일 수 있다.
- 구현:
  - 내부 액세스 키로 보호된 콘텐츠 스튜디오
  - 설명 가능한 기회 점수와 점수 근거
  - Gemini 구조화 출력 기반 대본·장면·CTA·고지 초안
  - 원문 출처와 게시 전 체크리스트
- 제외:
  - 외부 계정 생성
  - 자동 게시
  - 유료 광고
  - 실제 영상·TTS 렌더링
- 선행 지표:
  - 상위 10개 중 운영자가 제작 가치 있다고 평가한 비율 50% 이상
  - 초안 생성 성공률 95% 이상
  - 검수 전 사실 오류 5% 미만
  - 건당 기회 탐색+초안 시간 10분 이하
- 수익 연결 지표: 콘텐츠→사이트 CTR, 유효 재방문, 제작비/유효 방문
- 중단 기준: 후보 적합도 30% 미만 또는 사실 오류 10% 초과가 2회 연속 발생
- 검토일: 실제 초안 10건 검수 후
- 승인 지점: 어떤 초안도 사용자 승인 없이 외부에 게시하지 않는다.
- 2026-07-24 진행:
  - 콘텐츠 스튜디오에 후보 `제작 가치 있음/보류/제외`, 대본 `사용 가능/수정 필요/폐기` 검수 기록을 추가했다.
  - 검수 기록은 브라우저에 저장하며, E-001 표본 10건을 JSON으로 내보내 적합도와 수정 필요 비율을 계산할 수 있다.
  - 실제 10건 평가는 편집 판단이 필요하므로 완료로 처리하지 않는다.

## E-002 — 숏폼 포맷 수동 파일럿

- 상태: 대기
- 선행 조건: E-001 초안 10건 검수, 게시 계정과 플랫폼 약관 승인
- 가설: “숫자 변화 → 왜 중요한가 → 내일 확인할 지표” 구조가 단순 뉴스 요약보다 완주율과 사이트 CTR이 높다.
- 설계: 훅만 다른 3개 포맷을 각 3~4회 게시하고 주제·길이·게시 시간을 가능한 한 통제한다.
- 성공 기준 초안: 3초 유지율, 평균 시청 비율, 완주율, 사이트 CTR 중 2개 이상이 기존 기준 대비 20% 개선
- 비용 제한: 유료 광고 없이 유기적 도달로 먼저 검증
- 승인 지점: 계정 생성, 게시, AI 공개 설정은 사용자 승인 필요

## E-003 — 원본 설명 페이지와 광고 준비도

- 상태: 대기
- 선행 조건: 편집 정책·개인정보/쿠키 고지, 작성자/검토자 표시, 원본 설명 페이지 10개 이상
- 가설: 기사 링크 목록이 아니라 고유한 데이터 설명 페이지를 늘리면 검색 유입과 재방문이 증가하고 AdSense 심사 준비도가 높아진다.
- 측정: 검색 노출/CTR, 페이지당 참여시간, 다음 페이지 이동, 7일 재방문
- 광고 게이트: AdSense 계정·약관 승인 전에는 광고 코드를 운영 환경에 추가하지 않는다.

## E-004 — 출처가 분리된 관심 신호와 원본 유입 루프

- 상태: 개발·배포 검증 중
- 시작일: 2026-07-22
- 대상: 출퇴근 중 오늘의 금융 변수를 빠르게 이해하려는 20~40대 한국 개인 투자자
- 문제:
  - InvestBoard 내부 조회·댓글·투표와 원문 플랫폼의 반응이 같은 숫자처럼 보이면 인기도를 오해할 수 있다.
  - 단순 뉴스 목록은 검색 유입과 재방문을 만들 고유한 설명 가치가 부족하다.
  - 페이지별 canonical, 사이트맵, 공유 추적이 부족해 콘텐츠별 성과를 측정하기 어렵다.
- 근거와 확인일(2026-07-22):
  - 네이버 뉴스 검색 공식 API 응답은 제목·URL·요약·발행시각을 제공하지만 조회·댓글·추천 수는 제공하지 않는다.
  - `news.naver.com`, `n.news.naver.com`, `apis.naver.com` robots.txt는 일반 자동 수집을 `Disallow: /`로 차단한다.
  - 네이버 DataLab은 공식 API로 검색 관심도의 0~100 상대값을 제공한다.
  - Schema.org `InteractionCounter`는 공개 페이지가 조회·댓글·좋아요·싫어요를 표준 형식으로 제공할 수 있게 한다.
  - YouTube Data API는 공개 영상의 조회·좋아요·댓글 수를 제공하며 싫어요는 영상 소유자 외에는 비공개다.
- 가설:
  - 내부 반응, 원문 공개 반응, 검색 관심도를 출처별로 분리하고 원본 브리핑·가이드로 연결하면 신뢰를 해치지 않으면서 두 번째 행동과 재방문이 증가한다.
- 구현 범위:
  - robots 허용 페이지의 공개 Schema.org 참여 수치 수집
  - 선택적 YouTube 공식 API 수집과 네이버 DataLab 공식 카테고리 분야 검색 관심도
  - 외부 수치의 출처·상태·갱신시각 저장, 내부 수치와 분리 표시
  - 정책상 차단, 미지원, 키 미설정을 숫자 0으로 위장하지 않고 상태로 표시
  - 홈 `오늘 내 돈에 영향을 주는 3가지`, `/briefing`, 원본 가이드 4개
  - 페이지별 canonical, RSS, 사이트맵, 1200×630 공유 이미지
  - 공유 UTM, 50% 읽기, 도구 사용 이벤트
- 제외 범위:
  - 네이버 비공식 댓글·추천 내부 API 호출
  - robots 차단 우회, 로그인 세션 재사용, CAPTCHA 우회
  - 외부 댓글 원문 저장 또는 InvestBoard에서 외부 플랫폼으로 댓글 전송
  - 검색엔진 도메인 등록, 외부 채널 게시, 유료 광고
- 선행 지표:
  - 브리핑에서 기사·도구·가이드로 이어지는 두 번째 행동률 15% 이상
  - 원본 가이드 50% 읽기율 25% 이상
  - 공유 링크 유입 중 두 번째 행동률이 전체 신규 방문보다 높을 것
  - 외부 수치가 있는 기사에서 공급자와 내부/외부 구분 표시 누락 0건
- 수익 연결 지표: 검색·공유 기반 유효 방문, 7일 재방문, 회원/관심주제 저장 전환, 유효 방문당 콘텐츠·API 비용
- 예상 반복 비용:
  - Schema.org 수집은 자체 서버 네트워크·CPU 비용만 발생하며 15분당 최대 12건(시간당 최대 48건)으로 제한하고, 인기 신호가 높은 기사부터 처리한다.
  - 네이버 DataLab은 6시간마다 1회로 공식 일 1,000회 한도보다 낮게 운영한다.
  - YouTube API는 키가 있을 때 영상 URL에만 1-unit 조회를 사용한다.
- 위험과 대응:
  - 정책 변경: robots를 6시간 캐시 후 재확인하고 확인 실패 시 수집하지 않는다.
  - SSRF: 공개 HTTP(S), 기본 포트, 공인 IP만 허용하고 리다이렉트마다 재검증한다.
  - 숫자 오인: 원문·내부 수치를 합산하지 않고 검색 관심도를 실제 조회수로 표현하지 않는다.
  - 얇은 콘텐츠: 수집 기사 상세는 `noindex`, 원본 설명·계산·체크리스트만 사이트맵에 포함한다.
- 성공 기준: 배포 후 30일 내 원본 페이지 검색 노출 발생, 두 번째 행동률 15% 이상, 7일 유효 재방문 8% 이상 중 2개 이상 달성
- 중단 기준: 외부 수치 정확성 오류 5% 초과, 정책 차단 요청 발생, 원본 콘텐츠 10개 이후에도 유효 방문이 뉴스 목록 대비 개선되지 않음
- 검토일: 운영 데이터 30일 또는 유효 방문 500건 중 먼저 도달한 시점
- 승인 지점: 외부 계정·API 약관 추가, 게시, 광고, 유료 공급자 계약은 사용자 승인 전 실행하지 않는다.
- 2026-07-24 진행:
  - DataLab·외부 반응 수집기의 마지막 실행, 성공, 실패, 처리량과 오류를 DB에 저장하는 운영 진단 API를 구현했다.
  - 콘텐츠 스튜디오에서 키 노출 없이 DataLab 설정 여부, 값 보유 기사 수, 외부 실수치 수집률, 상태별 기사 수를 확인할 수 있게 했다.
  - `robots.txt`, `sitemap.xml`, `feed.xml`, EC2 health는 운영 환경에서 HTTP 200을 확인했다.
  - 공개 검색엔진의 `site:investboard.cloud`, `site:investboard.cloud/guides`, `site:investboard.cloud/briefing` 질의에는 결과가 없어 인덱싱 검증은 계속 진행한다.
