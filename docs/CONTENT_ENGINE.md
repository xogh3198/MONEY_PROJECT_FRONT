# InvestBoard 콘텐츠 엔진

마지막 검증일: 2026-07-22

## 목적

콘텐츠 엔진은 유행을 무작정 복제하는 자동 게시기가 아니다. 여러 데이터에서 시의성 있는 금융 질문을 발견하고, InvestBoard의 숫자·도구·설명을 더한 **검수 가능한 원본 콘텐츠 초안**을 만드는 시스템이다.

## 전체 흐름

```text
수집 → 정규화·출처 기록 → 금융 관련성 필터 → 기회 점수 → 원본 해설 초안
    → 사실·권리·정책 검수 → 영상/음성 렌더링 → 게시 승인 → 성과 수집 → 점수 보정
```

외부 게시와 유료 광고 집행은 항상 승인 단계 뒤에 둔다.

## 데이터 소스

### 현재 사용

- 네이버 뉴스 검색 API
- 경제 매체 RSS
- 네이버 랭킹 페이지의 인기 신호
- 코스피, 코스닥, 환율, S&P 500, 비트코인, 금 지표
- 사이트 내 조회·댓글·투표
- Vercel Analytics

### 다음 연결 후보

- [네이버 DataLab 검색어 트렌드 API](https://developers.naver.com/docs/datalab/search/): 한국 검색 관심도의 상대 변화
- [YouTube Data API `videos.list`](https://developers.google.com/youtube/v3/docs/videos/list): `chart=mostPopular`, `regionCode=KR` 기반 플랫폼 관심도
- Google Search Console: 실제 검색 노출·클릭·질문
- YouTube Analytics / Meta Insights: 게시 후 유지율·완주율·사이트 클릭

외부 플랫폼 메타데이터는 주제 신호로만 사용한다. 다른 제작자의 영상, 음성, 썸네일이나 대본을 다운로드·재사용하지 않는다.

## 기회 점수

MVP 점수는 설명 가능한 규칙 기반 점수다.

```text
기회 점수 = 금융 관련성 + 최신성 + 내부 인기 + 참여 + 시장 연결성 + 설명 가치 - 위험 패널티
```

- 금융 관련성: 금리, 환율, 주식, 물가, 주거비, 세금, 자산 등 핵심어
- 최신성: 48시간 안의 기사에 높은 점수
- 내부 인기: InvestBoard 실제 조회수의 로그 스케일
- 외부 트렌드: 외부 랭킹 위치를 정규화한 탐색 신호(원문 조회수·댓글·좋아요가 아님)
- 참여: 댓글과 투표
- 시장 연결성: 같은 시점의 비정상 변동과 연결 가능성
- 설명 가치: 사용자가 “왜?”, “나에게 어떤 영향?”을 물을 만한 소재
- 위험 패널티: 사실 근거 부족, 연예·범죄 등 금융 연결이 약한 소재, 공포·과장 유도, 개인 매수·매도 지시 위험

이 점수는 실제 대중 관심도의 확정값이 아니라 제작 후보를 정렬하는 탐색 지표다. 게시 성과가 쌓이면 가중치를 실험 단위로 보정한다.

## 콘텐츠 초안 규격

각 초안은 아래 필드를 가진다.

- 실험 ID와 제작 상태
- 원문 제목·매체·URL·수집/발행 시각
- 기회 점수와 점수 근거
- 타깃 시청자와 한 문장 가치 제안
- 1~2초 훅
- 45~60초 내레이션
- 9:16 장면별 화면 문구와 시각 자료 지시
- 사이트의 관련 지표·도구·설명 페이지 CTA
- 제목, 설명, 해시태그 후보
- 사실/추론 구분, 금융 고지, AI 사용 고지
- 권리·사실·브랜드·정책 검수 체크리스트

## 생성 원칙

- 기사 본문을 복사하거나 문장만 바꿔 재출판하지 않는다.
- 입력으로 제공된 사실 밖의 수치·인용·사건을 만들어내지 않는다.
- 불확실한 내용은 추론 또는 확인 필요로 표시한다.
- “무조건 오른다”, “지금 사라”, “수익 보장”을 금지한다.
- 정보의 의미와 한계를 함께 설명한다.
- 충격·공포만 유도하는 훅과 사실을 숨기는 클릭베이트를 금지한다.
- 금융상품·제휴·스폰서가 포함되면 경제적 이해관계를 눈에 띄게 표시한다.

## 영상과 음성

### 권장 제작 순서

1. 규칙 기반 기회 선정
2. AI 구조화 초안
3. 사람의 사실·권리 검수
4. 자체 차트·타이포그래피·라이선스 자산으로 장면 생성
5. 사용 권리가 확인된 음성 또는 본인 음성 복제로 내레이션
6. 자막, 출처, AI 고지, 금융 고지 삽입
7. 비공개 렌더 검수
8. 게시 승인

타인의 목소리를 허락 없이 복제하지 않는다. 현실적인 합성 장면이나 타인 음성 복제를 사용하면 YouTube의 altered content 및 Meta의 AI 정보 공개 절차를 따른다.

### 화면 포맷 기본값

- 9:16, 1080×1920
- 첫 1~2초에 질문 또는 숫자 변화 제시
- 한 장면에 한 메시지, 큰 자막
- 45~60초를 기본 실험 길이로 사용하되 성과에 따라 변경
- 마지막 CTA는 광고 클릭이 아니라 관련 데이터·계산기·설명 페이지 확인

## 게시 전 게이트

다음 항목이 모두 통과해야 게시 후보가 된다.

- [ ] 원문과 수치가 실제 출처와 일치한다.
- [ ] 사실과 분석/추론이 구분되어 있다.
- [ ] 사용한 이미지·영상·음성·음악의 권리가 기록되어 있다.
- [ ] 직접적인 투자 지시나 수익 보장이 없다.
- [ ] 제휴·스폰서·AI 사용 고지가 필요한 경우 포함됐다.
- [ ] 제목과 썸네일이 본문을 오도하지 않는다.
- [ ] 사이트 CTA가 실제로 유용한 관련 페이지로 연결된다.
- [ ] 실험 ID와 UTM이 지정됐다.
- [ ] 게시 계정과 공개 범위가 승인됐다.

## AI 모델 운영

- 텍스트 초안 기본 모델: `gemini-3.1-flash-lite`
- 모델명은 `GEMINI_MODEL` 환경변수로 교체 가능하게 유지한다.
- 2026-06-01 종료된 `gemini-2.0-flash-lite`를 사용하지 않는다.
- 모델 응답은 JSON 스키마와 애플리케이션 검증을 함께 사용한다.
- 입력·출력에 API 키, 사용자 토큰, 개인정보를 넣지 않는다.

공식 모델 상태는 [Gemini 모델 문서](https://ai.google.dev/gemini-api/docs/models)와 [지원 종료 일정](https://ai.google.dev/gemini-api/docs/deprecations)에서 배포 전 다시 확인한다.

## MVP 보안

콘텐츠 스튜디오는 `CONTENT_STUDIO_ACCESS_KEY`가 있어야 호출할 수 있는 내부 도구로 운영한다. 키는 브라우저 로컬 저장소와 HTTPS 요청 헤더에만 사용하고 소스, 로그, URL, 분석 이벤트에 넣지 않는다.

공개 자동 게시, 유료 렌더링, TTS, 광고 API는 첫 MVP 범위에서 제외한다. 실제 포맷 성과가 검증된 뒤 공급자 비용·권리·정책을 비교해 연결한다.
